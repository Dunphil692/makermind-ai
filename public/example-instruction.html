一次函数：停车挑战雷达

这是一个静态示例 instruction，用来展示 MakerMind AI 最终希望生成的课堂项目说明书效果。

Project Overview

项目概述

学生需要制作一个“停车挑战雷达”。超声波传感器检测小车或手掌距离， OLED 屏幕显示距离数值，LED 灯带根据距离变化显示不同颜色。 学生通过调整一次函数中的斜率和截距，理解输入 x 和输出 y 之间的关系。

Learning Goal

学习目标

  • 理解一次函数 y = kx + b 中 k 和 b 的作用
  • 知道传感器读数可以作为函数输入 x
  • 能解释输出 y 如何控制灯光、蜂鸣器或屏幕反馈
  • 能通过修改 k 和 b 改变作品的交互效果
Interaction Flow

交互流程

  1. 学生把手或小车靠近超声波传感器
  2. 传感器读取距离 x
  3. 程序计算 y = kx + b
  4. 系统根据 y 控制 LED 颜色、蜂鸣器频率或屏幕提示
  5. 学生修改 k 和 b，观察反馈变化
Materials

材料清单

  • Arduino 或 ESP32 主控板
  • 超声波传感器
  • LED 灯带或 RGB LED
  • 蜂鸣器
  • OLED 显示屏
  • 杜邦线、面包板、纸板结构件
Build Steps

制作步骤

  1. 连接超声波传感器，读取距离数据
  2. 把距离 x 显示在 OLED 屏幕上
  3. 设置一次函数 y = kx + b
  4. 用 y 控制 LED 颜色或蜂鸣器频率
  5. 制作停车区域外壳，让项目更接近真实场景
  6. 调整 k 和 b，测试不同距离下的反馈效果
Knowledge Explanation

知识点讲解

在这个项目中，距离 x 是输入，灯光亮度、蜂鸣器频率或屏幕提示 y 是输出。 当 k 变大时，距离变化会带来更明显的反馈变化；当 b 变化时，整个反馈范围会整体上移或下移。 学生不是只在纸上画函数图像，而是在真实硬件反馈中观察函数关系。

Starter Code

代码思路

C++ / Arduino 
float k = 1.5;
float b = 10;

void loop() {
  float x = readDistance();
  float y = k * x + b;

  showDistance(x);
  controlLight(y);
  delay(100);
}
MicroPython / K10 
k = 1.5
b = 10

while True:
    x = read_distance()
    y = k * x + b

    show_distance(x)
    control_light(y)
Practice

融会贯通训练

  • 如果希望车辆靠近时灯光变化更明显，应该调整 k 还是 b？为什么？
  • 如果希望一开始灯光就更亮，应该调整 k 还是 b？
  • 把 y = 1.5x + 10 改成 y = 3x + 5，观察反馈有什么不同
  • 尝试设计一个“安全距离警报区间”，并解释你的函数设计
FAQ

常见问题

学生只想玩灯光，不想解释函数怎么办？

可以要求学生每修改一次 k 或 b，都要记录一次现象，并用一句话解释为什么变化。

如果传感器读数不稳定怎么办？

可以让学生多读取几次取平均值，顺便引出数据噪声和测量误差。
