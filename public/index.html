Problem

学生不是不想学，而是缺少一个愿意学的入口

我们想解决的不是“怎么让网页更酷”，而是“怎么让学生愿意开始学，并且真的学会”。

01

知识太抽象

公式、概念和生活场景之间没有连接，学生不知道为什么要学。

02

学习过程太被动

学生先听讲、再记公式、再刷题，很难持续保持兴趣。

03

项目可能只剩“好玩”

有些项目看起来很热闹，但学生做完作品后，并没有真正理解背后的知识。

How it works

把知识点变成一个可以动手完成的学习项目

老师输入知识点、学生兴趣和可用材料，系统生成一份可以带着学生完成的动手学习方案。

1

输入知识点

例如一次函数、概率、电路、浮力、循环结构。

2

选择兴趣和材料

例如游戏、音乐、运动、Arduino、ESP32、K10 或纸板结构。

3

生成动手项目

系统给出项目故事、材料清单、制作步骤、知识讲解和代码思路。

4

边做边理解

学生在调试作品、改变参数、观察反馈的过程中理解知识。

5

最后验证学会

通过迁移挑战和解释题，检查学生能不能把知识用到新场景。

Example

一次函数，可以变成一个停车挑战雷达

把抽象公式放进真实反馈里，学生会更容易理解输入、输出、斜率和截距。

传统学习方式

学习 y = kx + b

  • 画函数图像
  • 代入数字计算
  • 判断斜率和截距
  • 完成练习题

很多学生会做题，但不知道这个公式和真实世界有什么关系。

MakerMind 方式

制作一个停车雷达

  • 超声波传感器读取距离 x
  • 程序计算 y = kx + b
  • y 控制 LED 灯光、蜂鸣器或屏幕提示
  • 学生修改 k 和 b，观察反馈变化

学生会自然理解：x 是输入，y 是输出，k 控制变化速度，b 控制初始状态。

Cases

不同知识点，都可以变成好玩的项目

我们希望学生从“这个知识点要背下来”，变成“原来这个知识可以让我的作品动起来”。

一次函数 →

停车雷达：用距离传感器和灯光反馈理解输入、输出、斜率和截距。

概率 →

投篮挑战器：通过命中次数、总次数和命中率理解概率和统计。

电路 →

反应训练按钮：通过按钮、LED 和蜂鸣器理解电路连接和信号控制。

声音频率 →

音乐灯光墙：通过声音输入和灯光变化理解频率、节奏和数据反馈。

条件判断 →

自动宠物喂食器：通过传感器数据和判断规则理解 if / else 逻辑。

Comparison

同一个知识点，可以有两种学习方式

传统方式

  • 老师先讲概念
  • 学生记公式
  • 通过刷题练习
  • 容易觉得知识枯燥
  • 做完题后很快忘记

MakerMind 方式

  • 学生先进入一个有趣任务
  • 在制作过程中遇到真实问题
  • 使用知识解决问题
  • 通过作品看到知识的作用
  • 最后用自己的话解释原理
